Windows Incident Response Cheat Sheet

Step 1: What is Incident Response?

Incident Response is the process of identifying, managing, and responding to security threats or attacks on a computer system or network. Security incidents could include unauthorized system access, malware infections, or network breaches.

Common security events include:

  • Unauthorized use of system privileges
  • System crashes or abnormal behavior
  • Malicious software (malware)

Step 2: Check User Accounts

Why it’s important: Monitoring user accounts helps identify any suspicious logins or account changes.

  • GUI Method: Press Windows + R, type lusrmgr.msc to see local user accounts.
  • Command Prompt: Open as Admin and type net user to view user accounts.
  • PowerShell: Run Get-LocalUser to see account details like enabled status and type.

Step 3: Monitor Running Processes

Why it’s important: Suspicious processes running on the system can indicate malicious activity.

  • Task Manager (GUI): Press Windows + R, then type taskmgr.exe to view all active processes.
  • Command Prompt: Run tasklist to see all processes and their details.
  • PowerShell: Type get-process to see running processes.
  • WMIC (Command Prompt/PowerShell): Use wmic process list full to find suspicious processes with abnormal behavior.

Step 4: Check Active Services

Why it’s important: Abnormal services could indicate unauthorized software running on the system.

  • GUI Method: Press Windows + R, type services.msc to see a list of services.
  • Command Prompt: Run net start to view all active services.
  • PowerShell: Use Get-Service to list running services.

Step 5: Review Task Scheduler

Why it’s important: Malicious programs might schedule tasks to run at specific times.

  • GUI Method: Navigate to C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Administrative Tools and open Task Scheduler.
  • Command Prompt: Run schtasks to list scheduled tasks.

Step 6: Investigate Startup Programs

Why it’s important: Malicious programs can be set to start automatically when the system boots.

  • Task Manager (GUI): Open Task Manager and go to the ‘Startup’ tab to see all startup programs.
  • PowerShell: Run Get-CimInstance Win32_StartupCommand to view all auto-start programs.

Step 7: Analyze Windows Registry

Why it’s important: Malware can modify the Windows registry to persist on reboot or alter system settings.

  • GUI Method: Open REGEDIT and navigate to the Run keys in the registry.
  • PowerShell: Run reg query HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run to check the local machine registry for suspicious entries.

Step 8: Check Active TCP & UDP Ports

Why it’s important: Open ports can provide attackers with entry points into the system.

  • Command Prompt: Run netstat -ano to view active network connections.
  • PowerShell: Use Get-NetTCPConnection to view active connections and their ports.

Step 9: Monitor File Sharing

Why it’s important: Unauthorized file sharing could expose sensitive data or open vulnerabilities.

  • Command Prompt: Run net view <localhost> to view shared folders.
  • PowerShell: Use Get-SMBShare to list shared resources.

Step 10: Inspect System Files

Why it’s important: Files with unusual extensions or modification dates could indicate malware or unauthorized changes.

  • Command Prompt: Run forfiles /D -10 /S /M *.exe to find recently modified .exe files.
  • PowerShell: Check the modification dates of files by running forfiles /D -10 /S /M *.exe.
